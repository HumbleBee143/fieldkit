# Field-Kit Tool Builder

## Mandatory File Structure

Every tool is a **single self-contained index.html file**. No exceptions.

```
public/
└── tool-name/
    └── index.html    ← everything goes here
```

- All CSS in a `<style>` block in the `<head>`
- All JS in a `<script>` block before `</body>`
- No separate `.css`, `.js`, or asset files
- No npm packages, no build step, no bundler
- No external JS libraries unless loaded from a CDN via `<script src>`

## Watermark (Required on All Tools)

Every tool must include a Field-Kit watermark on printed/exported output.

```html
<div class="watermark">Generated by Field-Kit — field-kit.co.uk</div>
```

```css
.watermark {
  text-align: center;
  font-size: 0.75rem;
  color: #999;
  margin-top: 2rem;
}

@media print {
  .watermark { display: block; }
}
```

The watermark is hidden on screen by default and visible on print. Do not remove it.

## Pro Unlock Pattern

Pro features (logo upload, remove watermark) are unlocked via a Gumroad licence key stored in localStorage.

```js
function isProUnlocked() {
  return localStorage.getItem('fieldkit_pro_key') === 'VALID_KEY';
}
```

- Gate logo upload and watermark removal behind `isProUnlocked()`
- Always ship with the watermark visible — Pro just hides it
- Do not implement payment logic — that is handled externally via Gumroad

## Print / PDF Export

All tools must be printable via `window.print()`.

```html
<button onclick="window.print()">Export PDF</button>
```

```css
@media print {
  button, .no-print { display: none; }
  body { background: white; color: black; }
  .watermark { display: block; }
}
```

## Adding a New Tool to the Homepage

When a new tool is built, add a card to `public/index.html`:

```html
<a href="/tool-name" class="tool-card">
  <h3>Tool Name</h3>
  <p>One-line description of what it does.</p>
</a>
```

## Deployment

- Commit to main branch — Cloudflare Pages auto-deploys
- No build command needed
- Publish directory: `public`
- Test locally by opening the HTML file directly in a browser

## Proof of Delivery (Required)

Before marking any task complete or posting a completion comment, you MUST run the following and include the output in your comment:

```bash
find public/tool-name -type f -not -path '*/node_modules/*'
```

The output must show `public/tool-name/index.html` exists. If it does not exist on disk, the task is NOT complete — do not mark it done or reassign it for review.

## Conventions Summary

- Single `index.html` per tool — always
- Watermark on all output — always
- Print export via `window.print()` — always
- No separate files, no build tools, no frameworks
- Pro unlock via localStorage key — gate logo upload and watermark removal only
- Proof of delivery required — always run file listing before closing a task
